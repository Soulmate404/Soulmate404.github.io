认识JDBC

JDBC英文名为：Java Data Base Connectivity（Java数据库连接）官方解释它是Java编程语言和广泛的数据库之间独立于数据库的连接标准的Java API，根本上说JDBC是一种规范，它提供的接口，一套完整的，允许便捷式访问底层数据库。所以说几乎所有数据库都支持通过JDBC进行访问，我们这里以MySQL作为例子

首先我们需要导入JDBC依赖。这有两种方式，你可以直接去MySQL官网上下载属于MySQL的JDBC依赖，也可以通过Maven直接导入依赖

还可以手动设置事务隔离等级

支持的事务等级如下

  1. Connection.TRANSACTION_NONE
    • 描述：不支持事务，每次操作都是独立的。
    • 适用场景：适用于不需要事务的简单操作。
  2. Connection.TRANSACTION_READ_UNCOMMITTED
    • 描述：读未提交。允许一个事务读取另一个事务未提交的数据，可能会出现脏读、不可重复读和幻读。
    • 适用场景：对数据一致性要求不高的场景，但并发性能较好。
  3. Connection.TRANSACTION_READ_COMMITTED
    • 描述：读已提交。一个事务只能读取另一个事务已经提交的数据，可以避免脏读，但可能会出现不可重复读和幻读。
    • 适用场景：适用于大多数普通的业务场景，平衡了性能和数据一致性。
  4. Connection.TRANSACTION_REPEATABLE_READ
    • 描述：可重复读。保证在同一个事务中，多次读取同一数据的结果是一致的，可以避免脏读和不可重复读，但可能会出现幻读。
    • 适用场景：适用于对数据一致性要求较高的场景，但并发性能稍差。
  5. Connection.TRANSACTION_SERIALIZABLE
    • 描述：可串行化。最高级别的隔离，事务完全隔离，避免了脏读、不可重复读和幻读。但并发性能最低。
    • 适用场景：适用于对数据一致性要求极高的场景，如金融交易系统。

其实呢关于JDBC还有很多进一步的应用,但不怎么常用,有些操作也存在安全性问题,所以这里就不过多介绍了,让我们直接进入MyBaits

Mybatis是什么

MyBatis 是一个优秀的 ORM（对象关系映射） 框架，它简化了 Java 程序对数据库的访问。相比于传统的 JDBC，MyBatis 提供了更灵活的 SQL 语句管理方式，并支持动态 SQL，极大地提高了数据库操作的开发效率。Mybatis的学习成本低,优化好,是大部分项目的持久层框架的最佳选择人
