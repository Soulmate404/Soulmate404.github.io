MySQL入门

MySQL入门

任何一个软件都会接受，产生，处理大量的数据，这些数据的种类各不相同，为了管理这些数据，我们创建了数据库（DateBase）这一概念并有了相应的数据库管理系统，进一步的创建出了一套复杂的数据库语言。本篇站在一个学习java的新手的角度，快速的过一遍与数据库相关的那些你应当掌握的基础内容

概念

  • 数据库：DataBase，简称DB。按照一定格式存储数据的一些文件的组合顾名思义: 存储数据的仓库，实际上就是一堆文件。这些文件中存储了具有特定格式的数据。

  • 数据库管理系统：DataBaseManagement，简称DBMS。数据库管理系统是专门用来管理数据库中数据的，数据库管理系统可以对数据库当中的数据进行增删改查。

    • 常见的数据库管理系统:
      MySQL、 OracleMS SqlServer、 DB2、 sybase等

  • SQL: 结构化查询语言程序员需要学习SQL语句，程序员通过编写SQL语句，然后DBMS负责执行SQL语句，最终来完成数据库中数据的增删改查操作。

  • 以上三者关系：

    DBMS –(执行)–> SQL –(操作)–> DB

枚举和集合类型

1. ENUM（枚举）类型

ENUM是一种枚举类型，用于存储一组预定义的字符串值。每个ENUM字段只能存储一个值，这些值必须在定义时预先指定。

语法：

特点

  1. 存储效率：ENUM类型在存储时非常高效。MySQL会将每个枚举值映射为一个整数（从1开始），实际存储的是这个整数，而不是字符串本身。
  2. 默认值：如果插入的值不在预定义的枚举列表中，MySQL会存储空字符串''，并抛出警告。
  3. 排序：ENUM类型的值按照定义时的顺序进行排序，而不是按照字符串的字典顺序。

2. SET（集合）类型

定义

SET类型用于存储一组预定义的字符串值，但与ENUM不同的是，SET字段可以存储多个值（最多64个）。每个值必须在定义时预先指定。

语法：

特点

  1. 存储效率：SET类型在存储时也非常高效。MySQL会将每个值映射为一个位（bit），实际存储的是一个整数（位掩码）。
  2. 默认值：如果插入的值不在预定义的集合中，MySQL会存储空字符串''，并抛出警告。
  3. 排序：SET类型的值按照定义时的顺序进行排序。

默认约束

检查性约束

有些数据时有一个合理范围的,例如一个人的年龄不可能为一万岁,我们可以规定要求某些数据在插入时先检查是否符合要求(检查性约束在MySQL8.0.16以后支持),我们这里要求id不得大于100000,当然一些更复杂的运算也是支持的

主键约束

主键约束是一种特殊的唯一性约束,要求每张表只有一个主键且主键不能为空不能重复,作为每条数据的唯一索引,一般情况下每张表都应当由一个主键,这能大大加快查询的速度.主键可以选择与实际业务相关的值也可以选择与实际业务无关的值

例如我们选择将id作为主键

此外,也允许复合主键也就是多个字段共同作为主键的情况出现

此时复合主键和唯一性约束的规则相同,单个字段允许重复,多个字段的组合不允许重复

这里再提供两种简单的主键规则

  1. 自增主键,我们可以选择让主键自增。即从某个数开始每插入一条数据就在上一条数据的基础上加1,可以使用auto_increment实现

    末尾的auto_increment=100指从100开始自增,也可以不添加,默认从1开始

  2. UUID,或者说通用唯一标识符,这是由MySQL提供的一种生成一个36位(包含四个-)的随机字符串作为标识的方式,这一标识是完全随机且绝对不会重复的,可以作为每条数据的唯一标识,获取UUID可以使用MySQL内置的UUID函数UUID(),具体的使用我们后面再继续

外键约束是建立在两张表之间的,外键将一张表(称为子表或从表)某一字段与另一张表(称为父表或主表)的主键相关联,达成特定的效果

最基础的外键作用是保证数据的正确性,例如我们再建立一张表class,那么每个学生加入的班级必须是class表中存在的班级

外键约束还能实现以下的效果

  1. on delete cascade

    当主表中的记录被删除时从表中相关的记录一并被删除,如某个班毕业时只需删除主表中的班,学生也会自动删除

  2. on delete set null

    当主表中的某一字段被删除时从表中对弈字段被设置为null

  3. on update cascade

    当主表中该字段更新时对应字段也发生更新

  4. on delete restrict与on update restrict

    这两个选项是MySQL的默认行为,即当从表中有依赖的记录时不允许更新或删除主表中的对应记录,只有在规定了上面提到的三种限制时才能正常的改变从表中的记录

索引

当数据量达到一定程度时,每次查询都会耗费大量时间,我们可以通过建立索引来加速查询

在一般情况下执行查询操作,会遍历每一条数据用来对比数据是否满足条件,当对一个字段创建索引时,数据库会创建一个结合B+树与哈希表的结构来保存每一行数据的指针,当索引存在时执行查询操作会从该结构中获得对应数据的指针,再通过该指针直接读取数据,

注意,只有查询条件是对有索引的列进行判断时才有加速效果,具体的语法如下

索引类型 SQL 语法 作用 适用场景
单列索引 CREATE INDEX index_name ON table_name (column_name); 在单一列上创建索引，提升该列的查询速度 单列查询，如 WHERE column_name = 'value'
复合索引 CREATE INDEX index_name ON table_name (column1, column2, ...); 在多个列上创建索引，优化多列联合查询 多列查询，如 WHERE column1 = 'value1' AND column2 = 'value2'
唯一索引 CREATE UNIQUE INDEX index_name ON table_name (column_name); 确保该列的值唯一，不允许重复 数据完整性约束，如邮箱、用户名等唯一字段
主键索引 ALTER TABLE table_name ADD PRIMARY KEY (column_name); 在表上创建主键索引，保证列唯一且不允许 NULL 表的主键字段，如用户ID等
全文索引 CREATE FULLTEXT INDEX index_name ON table_name (column_name); 提供文本数据的全文搜索功能 大文本字段的搜索，如文章内容、博客评论等
空间索引 CREATE SPATIAL INDEX index_name ON table_name (column_name); 用于空间数据（如经纬度）的查询优化 地理信息系统（GIS）数据
索引删除 DROP INDEX index_name ON table_name; 删除现有索引 删除不再需要的索引
修改列索引 ALTER TABLE table_name ADD INDEX index_name (column_name); 为已有列添加索引 为已有表添加索引

事务机制

MySQL中存在事务机制,什么是事务机制,简单理解就是为实现某一目的而进行的对数据的操作,这个操作可以由多个sql语句组成,确保一批次的sql语句要么全部执行成功,要么全部失败,避免部分数据变动而另一部分数据没有变动造成的异常

一般来说,数据库的事务要满足以下几个原则

  • 原子性：一个事务（transaction）中的所有操作，要么全部完成，要么全部不完成，不会结束在中间某个环节。事务在执行过程中发生错误，会被回滚（Rollback）到事务开始前的状态，就像这个事务从来没有执行过一样。
  • 一致性：在事务开始之前和事务结束以后，数据库的完整性没有被破坏。这表示写入的资料必须完全符合所有的预设规则，这包含资料的精确度、串联性以及后续数据库可以自发性地完成预定的工作。
  • 隔离性：数据库允许多个并发事务同时对其数据进行读写和修改的能力，隔离性可以防止多个事务并发执行时由于交叉执行而导致数据的不一致。事务隔离分为不同级别，包括读未提交（Read uncommitted）、读提交（read committed）、可重复读（repeatable read）和串行化（Serializable）。
  • 持久性：事务处理结束后，对数据的修改就是永久的，即便系统故障也不会丢失。

此外,由于实际上一个数据库是多线程的,我们还要引入事务的隔离机制,避免多个事务间的相互干扰,具体的事务隔离级别如下

  • 读未提交（Read Uncommitted）：在此级别下，一个事务可以读取到其他事务尚未提交的数据。这可能导致脏读（Dirty Read），即读取到未提交事务的中间数据。

  • 读已提交（Read Committed）：在此级别下，一个事务只能读取到其他事务已经提交的数据。这解决了脏读问题，但可能会发生不可重复读（Non-repeatable Read），即同一查询在事务中执行多次，结果可能不同。

  • 可重复读（Repeatable Read）：在此级别下，事务中执行的查询将始终返回相同的结果，即使其他事务修改了数据。这解决了脏读和不可重复读问题。但仍可能发生幻读（Phantom Read），即在事务执行过程中，其他事务插入了新的数据行，导致查询结果发生变化。(默认状态)

  • 串行化（Serializable）：这是最高的隔离级别，事务会被完全隔离。它通过强制事务串行执行来避免脏读、不可重复读和幻读，但性能较差。

名词解释如下

  • 脏读（Dirty Read）：事务 A 修改了数据，但尚未提交，事务 B 读取了该数据，如果事务 A 回滚，事务 B 就读取了无效的数据。

  • 不可重复读（Non-repeatable Read）：事务 A 在执行过程中读取某个数据，事务 B 修改了该数据并提交，事务 A 再次读取该数据时，得到的结果与第一次读取时不同。

  • 幻读（Phantom Read）：事务 A 在查询时根据某个条件读取了若干条记录，事务 B 插入了符合条件的新数据，当事务 A 再次查询时，得到的记录数不同。

我们可以根据实际的需求来设置事务的隔离级别

可以在全局或者会话级别进行设置：

  • 全局级别

    会影响数据库中所有连接的事务。

  • 会话级别

    仅影响当前会话的事务。

NULL值的处理

在MySQL中NULL是一个极为特殊的值,任何值与NULL运算或者比较只能得到NULL,而不是真伪或者某个值

所以为了避免NULL锁造成的错误,除了可以用上面的ifnull函数外开可以使用

  • IS NULL: 当列的值是 NULL,此运算符返回 true。
  • IS NOT NULL: 当列的值不为 NULL, 运算符返回 true。
  • <=>: 比较操作符（不同于 = 运算符），当比较的的两个值相等或者都为 NULL 时返回 true。
