接线来我们来了解SpringMVC，这是一套用来实现与前端交互的框架，可以极为方便的帮我们写一些Web应用。不过，在此之前我们需要先了解另一个东西：TomCat，汤姆猫。它可以帮助我们快速的部署我们的Web项目。我这里贴一个官网Apache Tomcat® - Welcome!建议直接使用官方提供的Installer，可以直接帮我们跳过环境配置

ps:如果你没有前端基础,这将是非常痛苦的一节课,请努力撑下去

TomCat的简单介绍

首先我们需要启动它，在下载目录的bin中有一个startup.sh文件,运行这个文件即可,然后试着访问http://localhost:8080若能够成功访问则说明安装成功

稍微介绍一下各个文件夹的功能

  • lib目录：Tomcat服务端运行的一些依赖，不用关心。
  • logs目录：所有的日志信息都在这里。
  • temp目录：存放运行时产生的一些临时文件，不用关心。
  • work目录：工作目录，Tomcat会将jsp文件转换为java文件（我们后面会讲到，这里暂时不提及）
  • webapp目录：所有的Web项目都在这里，每个文件夹都是一个Web应用程序

此外我们还可以访问/example和/manager来查看自带的示例项目和管理页面.

如果现在没什么异常我们直接进入下一步

通过method来限制请求方式,请求方式如下,

请求方法描述
GET从服务器获取资源，例如网页、图片、视频等。GET请求不会对服务器资源进行修改。
POST向服务器发送数据以创建新资源或处理数据，例如提交表单或上传文件。数据包含在请求体中。
PUT更新服务器上的现有资源或创建新资源。PUT请求是幂等的，即多次请求结果相同。
DELETE删除服务器上的指定资源，也是幂等的。
PATCH对资源进行部分更新，仅修改资源的一部分内容。
HEAD类似于GET请求，但只返回响应头，不返回响应体。
OPTIONS查询服务器支持的HTTP请求方法。
CONNECT建立到服务器的隧道连接，通常用于代理或HTTPS连接。
TRACE回显服务器收到的请求，主要用于诊断。

Cookie与Session

接下来我们来了解一下Cookie与Session的使用.请思考一个问题.一个网站同一时间有成百上千的人同时访问,服务器该如何分得清这些人是谁呢?为什么我们在登录后服务器能一直知道我们的身份呢?这时你可能会想到端口相关的知识,但实际上http的端口为80,也就是说只有一个端口,那么我们该如何处理呢.这就用到了Cookie.Cookie是保存在浏览器中的一组数据,当向服务器发送请求时浏览器会附上这部分内容,而Cookie中又保存了每个用户的唯一标识,正是通过这种手段我们才能实现让服务器分得清到底是哪位用户在发送请求

一个Cookie包含：

  • name - Cookie的名称，Cookie一旦创建，名称便不可更改
  • value - Cookie的值，如果值为Unicode字符，需要为字符编码。如果为二进制数据，则需要使用BASE64编码
  • maxAge - Cookie失效的时间，单位秒。如果为正数，则该Cookie在maxAge秒后失效。如果为负数，该Cookie为临时Cookie，关闭浏览器即失效，浏览器也不会以任何形式保存该Cookie。如果为0，表示删除该Cookie。默认为-1。
  • secure - 该Cookie是否仅被使用安全协议传输。安全协议。安全协议有HTTPS，SSL等，在网络上传输数据之前先将数据加密。默认为false。
  • path - Cookie的使用路径。如果设置为“/sessionWeb/”，则只有contextPath为“/sessionWeb”的程序可以访问该Cookie。如果设置为“/”，则本域名下contextPath都可以访问该Cookie。注意最后一个字符必须为“/”。
  • domain - 可以访问该Cookie的域名。如果设置为“.google.com”，则所有以“google.com”结尾的域名都可以访问该Cookie。注意第一个字符必须为“.”。
  • comment - 该Cookie的用处说明，浏览器显示Cookie信息的时候显示该说明。
  • version - Cookie使用的版本号。0表示遵循Netscape的Cookie规范，1表示遵循W3C的RFC 2109规范

重定向与请求转发

当我们希望用户登录完成之后，直接跳转到网站的首页，那么这个时候，我们就可以使用重定向来完成。

那么我们接着来看请求转发，请求转发其实是一种服务器内部的跳转机制，我们知道，重定向会使得浏览器去重新请求一个页面，而请求转发则是服务器内部进行跳转，它的目的是，直接将本次请求转发给其他Servlet进行处理，并由其他Servlet来返回结果，因此它是在进行内部的转发。
