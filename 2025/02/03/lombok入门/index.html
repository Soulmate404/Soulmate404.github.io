lombok入门

lombok是一款用于简化代码的工具，能直接基于我们的代码生成一些对应的常用方法。当然，这个东西有些时候的风评其实不怎么好，会出现各种各样的奇奇怪怪的问题。但是，谁也无法否认的事实是这玩意的有些功能用起来时真的爽

首先 通过Maven引入

类相关

首先是与类相关的一些用法,包括自动生成对应的get,set方法以及构造方法等

@Getter与@Setter

@Getter注解用在类上可以自动生成类内所有属性的get方法,用在对应的属性上则只生成该属性的get方法

lombok会自动的生成所有属性的get方法

方法的权限默认为public,但可以主动设置

当然也可以细化到每个属性的权限

提供的选项包括

  • PUBLIC - 对应public关键字
  • PACKAGE - 相当于不添加任何访问权限关键字
  • PRIVATE - 对应private关键字
  • PROTECTED - 对应protected关键字
  • MODULE - 仅限模块内使用，与PACKAGE类似，相当于不添加任何访问权限关键字
  • NONE - 表示不生成对应的方法，这很适合对类中不需要生成的字段进行排除

@Setter注解则可以自动生成属性的set方法

如果我们自己手动编写了其中的某个方法,那么lombok会跳过,不会覆盖我们的方法

@Date与@Value

上面的几个注解如果堆在一个类上方,不怎么好看,所以我们还有一个@Date注解,这个注解等价于@Getter @Setter @RequiredArgsConstructor @ToString @EqualsAndHashCode

此外,如果这个类的对象的值是不希望被修改的,可以使用@Value

其等价于@Getter @FieldDefaults(makeFinal=true, level=AccessLevel.PRIVATE) @AllArgsConstructor @ToString @EqualsAndHashCode
